Yes — this is very doable in Google Sheets, but the Excel analogy is split across a few different trigger types. In Sheets/App Script terms, `onEdit(e)` fires when a user manually edits cells, while a **form submission** to a spreadsheet has its own trigger: `onFormSubmit()`. For a Google Form feeding rows into your sheet, **form submit** is the right hook for setting a new booking’s Status to `Pending`. ([Google for Developers][1])

One important gotcha: if your **Admin** sheet is being populated by `ARRAYFORMULA`, that formula spills its results across neighboring cells. In practice, that means you should **not** try to write your Status into the spill range itself. Put `Status` in its own separate, non-spilled column, or write it directly onto the actual form-response sheet instead. ([Google Help][2])

Here’s the cleanest version for your setup.

### Recommended approach

If the Google Form writes into **Form Responses 3** and that sheet contains a `Status` column, use this Apps Script:

```javascript
function setPendingOnFormSubmit(e) {
  const sheet = e.range.getSheet();

  // Change this to the sheet that actually receives the Google Form rows
  if (sheet.getName() !== 'Form Responses 3') return;

  const headers = sheet.getRange(1, 1, 1, sheet.getLastColumn()).getValues()[0];
  const statusCol = headers.indexOf('Status') + 1;

  if (statusCol === 0) {
    throw new Error('Could not find a column called "Status".');
  }

  const row = e.range.getRow();
  const statusCell = sheet.getRange(row, statusCol);

  // Only set it if blank
  if (!statusCell.getValue()) {
    statusCell.setValue('Pending');
  }
}
```

1. Open the spreadsheet.
2. Go to **Extensions → Apps Script**.
3. Paste the code in.
4. Save.
5. In Apps Script, go to **Triggers**.
6. Add a trigger for `setPendingOnFormSubmit`

   * Event source: **From spreadsheet**
   * Event type: **On form submit**

That uses the spreadsheet’s installable **form submit** trigger, which is specifically meant for rows added by a linked Google Form. ([Google for Developers][3])

### If your Admin sheet is the one you care about

If **Admin** is only a mirrored view created by formula, then I would do this instead:

* keep columns A:G as your spilled form data
* make **H = Status**
* let the script write to **H** on the real response row, or map it separately in Admin

If you want, you can also maintain Status entirely on **Admin** as a standalone column keyed by row, but the simplest and strongest design is usually:

* **raw response data** on the form response sheet
* **Status** stored there too
* **Admin** just displays what you need

### If you also want Excel-like “change it back to Pending when edited”

That is a second behaviour. For example: if someone edits the date, time, or name later, you may want Status to go back to `Pending` unless it’s already `Completed` or something similar. That would use an edit trigger:

```javascript
function onEdit(e) {
  const sheet = e.range.getSheet();
  if (sheet.getName() !== 'Form Responses 3') return;

  const row = e.range.getRow();
  if (row === 1) return; // ignore header row

  const headers = sheet.getRange(1, 1, 1, sheet.getLastColumn()).getValues()[0];
  const statusCol = headers.indexOf('Status') + 1;
  if (statusCol === 0) return;

  // Don't react when the Status cell itself is edited
  if (e.range.getColumn() === statusCol) return;

  const statusCell = sheet.getRange(row, statusCol);
  const current = statusCell.getValue();

  // Adjust these rules however you like
  if (current !== 'Completed' && current !== 'Cancelled') {
    statusCell.setValue('Pending');
  }
}
```

That one behaves more like Excel’s `WorksheetChange`, but only for manual edits. `onEdit` is not the right mechanism for newly submitted Google Form rows, so the nicest setup is often:

* `setPendingOnFormSubmit(e)` for new bookings
* `onEdit(e)` for later manual changes ([Google for Developers][1])

The part I’d do next, in your place, is this: add a real `Status` column to the **actual response sheet** and let the script stamp `Pending` there. That will save you a lot of grief later 🌷

Perfect — for **column I**, I would **not** use Apps Script at all unless you absolutely have to. Because `I` is derived entirely from `G` and `H`, the clean Google Sheets way is to put **one** `ARRAYFORMULA` in `I2`, and let it fill itself for every new row that appears. `ARRAYFORMULA` is specifically meant to return results across multiple rows, so you do not need to “insert a new formula” row by row. ([Google Help][1])

Put this in **I2**:

```gs
=ARRAYFORMULA(IF((G2:G<>"")*(H2:H<>""), H2:H-G2:G, ))
```

That will:

* stay blank until both `G` and `H` have values
* calculate `H - G` for every populated row
* automatically cover future form entries as new rows arrive

The one rule is: **leave I3:I completely empty** apart from the spilled results, because array formulas need room to expand down the column. ([Google Help][1])

If those start/end values are times of day, the result is a **time duration**, so after adding the formula, format column **I** as a duration/time-style number format via **Format → Number** (or use a custom number format if you want a particular display). Google Sheets supports changing number and time formats from that menu. ([Google Help][2])

### Two useful variants

If you want **duration**, such as `1:30` for one and a half hours:

```gs
=ARRAYFORMULA(IF((G2:G<>"")*(H2:H<>""), H2:H-G2:G, ))
```

If you want **decimal hours for billing**, such as `1.5` instead of `1:30`:

```gs
=ARRAYFORMULA(IF((G2:G<>"")*(H2:H<>""), 24*(H2:H-G2:G), ))
```

In that second case, format column `I` as a normal number, not as time. The `24*` works because Sheets stores times as fractions of a day. ([Google Help][2])

### If a booking could cross midnight

Use this version instead:

```gs
=ARRAYFORMULA(IF((G2:G<>"")*(H2:H<>""), MOD(H2:H-G2:G,1), ))
```

That prevents a negative duration if something starts late in the evening and ends after midnight.

### For your `Status` column `J`

That one is different, because it is not a calculation from other columns — it is a value you want defaulted to `Pending`. That is where Apps Script with an `onFormSubmit` trigger still makes sense, because Apps Script can run automatically when a form response lands in the sheet. ([Google for Developers][3])

So the neat split is:

* **Column I** → formula only, with `ARRAYFORMULA`
* **Column J** → Apps Script sets default to `Pending`

That is the most Google-Sheets-ish way to do it 🌷

```javascript
function setPendingOnFormSubmit(e) {
  const ADMIN_SHEET_NAME = 'Admin';
  const STATUS_COL = 10; // Column J

  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const adminSheet = ss.getSheetByName(ADMIN_SHEET_NAME);

  if (!adminSheet) {
    throw new Error("Sheet 'Admin' not found.");
  }

  // Row number of the new form submission on the linked response sheet
  const row = e.range.getRow();

  // Ignore header row just in case
  if (row < 2) return;

  const statusCell = adminSheet.getRange(row, STATUS_COL);

  // Only set it if blank, so we don't overwrite anything manually changed later
  if (statusCell.isBlank()) {
    statusCell.setValue('Pending');
  }
}
```

Your script needs **two installable triggers**, not one: an **on form submit** trigger for `setPendingOnFormSubmit`, and an **on edit** trigger for `onApprovalEdit`. Your conflict logic is sound for a single calendar with two rooms. `calendar.getEvents(start, end)` returns events that start in, end in, or encompass that range, so checking those returned events for matching `location` is a reasonable way to detect same-room overlap. `createEvent(title, start, end, options)` supports both `location` and `description`, and `getId()` gives you the event’s ID to store back in the sheet. Use this:

```javascript
function combineDateAndTime_(datePart, timePart) {
  const dateObj = parseDatePart_(datePart);
  const timeObj = parseTimePart_(timePart);

  if (!dateObj || !timeObj) {
    return new Date('invalid');
  }

  return new Date(
    dateObj.year,
    dateObj.month,
    dateObj.day,
    timeObj.hours,
    timeObj.minutes,
    0,
    0
  );
}

function parseDatePart_(value) {
  if (value instanceof Date && !isNaN(value.getTime())) {
    return {
      year: value.getFullYear(),
      month: value.getMonth(),
      day: value.getDate()
    };
  }

  const d = new Date(value);
  if (d instanceof Date && !isNaN(d.getTime())) {
    return {
      year: d.getFullYear(),
      month: d.getMonth(),
      day: d.getDate()
    };
  }

  return null;
}

function parseTimePart_(value) {
  // Case 1: already a Date object
  if (value instanceof Date && !isNaN(value.getTime())) {
    return {
      hours: value.getHours(),
      minutes: value.getMinutes()
    };
  }

  // Case 2: text like "08:00" or "8:30"
  if (typeof value === 'string') {
    const m = value.trim().match(/^(\d{1,2}):(\d{2})$/);
    if (!m) return null;

    const hours = Number(m[1]);
    const minutes = Number(m[2]);

    if (hours < 0 || hours > 23 || minutes < 0 || minutes > 59) return null;

    return { hours, minutes };
  }

  // Case 3: numeric spreadsheet fraction of a day
  if (typeof value === 'number' && !isNaN(value)) {
    const totalMinutes = Math.round(value * 24 * 60);
    const hours = Math.floor(totalMinutes / 60) % 24;
    const minutes = totalMinutes % 60;
    return { hours, minutes };
  }

  return null;
}
```

```javascript
function ensureColumnsExist() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName(CONFIG.SHEET_NAME);
  if (!sheet) throw new Error('Sheet not found.');

  const headers = getHeaders_(sheet);
  const required = ['Status', 'Calendar Event ID', 'Processing Note'];
  let lastCol = sheet.getLastColumn();

  required.forEach(name => {
    if (!headers[name]) {
      lastCol += 1;
      sheet.getRange(1, lastCol).setValue(name);
    }
  });

  const refreshed = getHeaders_(sheet);

  const statusCol = refreshed['Status'];
  const assignedRoomCol = refreshed['Assigned Room'];

  const statusRule = SpreadsheetApp.newDataValidation()
    .requireValueInList([
      CONFIG.STATUS_VALUES.PENDING,
      CONFIG.STATUS_VALUES.APPROVED,
      CONFIG.STATUS_VALUES.REJECTED,
      CONFIG.STATUS_VALUES.CONFLICT
    ], true)
    .setAllowInvalid(false)
    .build();

  sheet.getRange(2, statusCol, Math.max(sheet.getMaxRows() - 1, 1), 1)
    .setDataValidation(statusRule);

  if (assignedRoomCol) {
    const roomRule = SpreadsheetApp.newDataValidation()
      .requireValueInList(['Studio', 'Gallery'], true)
      .setAllowInvalid(false)
      .build();

    sheet.getRange(2, assignedRoomCol, Math.max(sheet.getMaxRows() - 1, 1), 1)
      .setDataValidation(roomRule);
  }
}
```

### My recommendation

If a **material booking detail** is changed on an **Approved** row — and `Assigned Room` definitely counts — then the system should:

1. **Delete the existing calendar event**
2. **Clear `Calendar Event ID`**
3. **Set `Status` back to `Pending`**
4. **Write a clear `Processing Note`**
5. Then wait for the admin to review and approve again

### Why this is the cleanest approach

Because anything else creates drift.

If you only change `Approved` to `Pending` but leave the old calendar event in place, then:

* the calendar no longer matches the sheet
* the old room remains blocked
* the admin may think the booking is not live when in fact it still is
* you can accidentally create duplicates later

If you let the admin manage the calendar manually, you create **two sources of truth**:

* the sheet
* the calendar

That is exactly the sort of thing that causes “but I changed it over here” chaos later.

And while **auto-updating** the event in place is possible, I would *not* do that yet during your freeze. It skips the deliberate re-approval step, which is one of your main safeguards.

So for now:

**approved booking changed materially = old approval is invalidated**

That is tidy, logical, and safe.

> Once a booking is Approved, changing Assigned Room means the approval no longer stands.
> The existing calendar booking is removed, and the row returns to Pending so it can be reviewed again.

### The shape of the code

I would broaden your current edit handler so it can respond to two kinds of edits:

* `Status` changed to `Approved` → create booking
* `Assigned Room` changed on an already `Approved` booking → invalidate approval and remove old event

Something like this:

```javascript
function onAdminEdit(e) {
  if (!e || !e.range) return;

  const sheet = e.range.getSheet();
  if (sheet.getName() !== CONFIG.SHEET_NAME) return;

  const row = e.range.getRow();
  const col = e.range.getColumn();
  if (row < 2) return;

  const headers = getHeaders_(sheet);
  const statusCol = headers['Status'];
  const assignedRoomCol = headers['Assigned Room'];

  if (col === statusCol) {
    const newStatus = String(e.range.getValue()).trim();
    if (newStatus === CONFIG.STATUS_VALUES.APPROVED) {
      processApprovalRow_(sheet, row, headers);
    }
    return;
  }

  if (assignedRoomCol && col === assignedRoomCol) {
    handleApprovedBookingChange_(sheet, row, headers, 'Assigned Room');
    return;
  }
}
```

And then:

```javascript
function handleApprovedBookingChange_(sheet, row, headers, fieldName) {
  const statusCol = headers['Status'];
  const noteCol = headers['Processing Note'];
  const eventIdCol = headers['Calendar Event ID'];

  const currentStatus = String(sheet.getRange(row, statusCol).getValue()).trim();
  if (currentStatus !== CONFIG.STATUS_VALUES.APPROVED) return;

  const existingEventId = String(sheet.getRange(row, eventIdCol).getValue() || '').trim();
  let note = `${fieldName} changed after approval. Booking returned to Pending for re-review.`;

  if (existingEventId) {
    const calendar = getTargetCalendar_();
    const event = calendar.getEventById(existingEventId);

    if (event) {
      event.deleteEvent();
      sheet.getRange(row, eventIdCol).clearContent();
      note = `${fieldName} changed after approval. Previous calendar event deleted. Booking returned to Pending for re-review.`;
    } else {
      note = `${fieldName} changed after approval. Previous calendar event was not found; check calendar manually. Booking returned to Pending.`;
    }
  }

  sheet.getRange(row, statusCol).setValue(CONFIG.STATUS_VALUES.PENDING);
  sheet.getRange(row, noteCol).setValue(note);
}
```

Then your trigger should point to `onAdminEdit` rather than `onApprovalEdit`.

That is a very clean mental model for the admin.
